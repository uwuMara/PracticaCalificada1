# Sistema Distribuido de Gestión de Pedidos

##  Descripción
Este proyecto implementa un sistema distribuido basado en arquitectura cliente-servidor para la gestión de productos y pedidos.

El sistema está compuesto por:
- Cliente en Python con interfaz gráfica.
- Servidor en Java que actúa como middleware.
- Base de datos MySQL para persistencia de datos.

##  Arquitectura
Cliente (Python) → Servidor (Java) → Base de Datos (MySQL)

## ⚙️ Tecnologías utilizadas
- Python (Tkinter, sockets)
- Java (ServerSocket, JDBC)
- MySQL

##  Estructura del proyecto
- /servidor-java → Lógica del servidor y conexión a la base de datos.
- /cliente-python → Interfaz gráfica y cliente.
- base-datos/ → Script de la base de datos
